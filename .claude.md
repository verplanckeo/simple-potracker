# Project Overview

A modern web application built with React TypeScript frontend and .NET Web API backend.

## Architecture

- **Frontend**: React 18 + TypeScript + Vite
- **Backend**: .NET Web API
- **Styling**: Tailwind CSS + shadcn/ui
- **State Management**: React Context API / Zustand (specify when implemented)
- **Testing**: Vitest + React Testing Library
- **Build Tool**: Vite

## Project Structure

```
project-root/
├── frontend/              # React TypeScript application
│   ├── src/
│   │   ├── components/    # Reusable UI components
│   │   ├── pages/        # Page/route components
│   │   ├── hooks/        # Custom React hooks

```

## Development Workflow

When working on this project, Claude should:

1. Read and apply rules from `.claude/rules/` directory
2. Use TypeScript strictly - no `any` types
3. Follow React best practices and hooks patterns
4. Ensure accessibility (WCAG compliance)
5. Write tests alongside implementation
6. Use semantic commits

## Quick Commands

```bash
# Frontend development
cd frontend
npm install
npm run dev          # Start dev server
npm run build        # Production build (outputs to /dist)
npm run type-check   # TypeScript type checking without emit
npm run test         # Run tests
npm run lint         # Run ESLint

# Backend development
cd backend
dotnet restore
dotnet run          # Start API server
dotnet test         # Run tests
```

## Build Configuration

- **TypeScript Output**: All compiled JavaScript files go to `./dist` directory
- **Source Files**: Keep all `.ts` and `.tsx` files in `./src`
- **Never commit**: `dist/`, `*.js`, `*.d.ts` files (except config files)
- **Type Checking**: Use `npm run type-check` for validation without emitting files
